# Module 8: Put It to Work - Prepare for Cybersecurity Jobs

## Goal
Understand how to organize cybersecurity learning, explain skills clearly, and prepare for entry-level cybersecurity opportunities.

## Key Ideas

- **Transferable skills:** Skills from past work that apply to cybersecurity, such as troubleshooting, documentation, customer support, system administration, and communication.
- **Technical skills:** Hands-on abilities with tools, systems, logs, networks, operating systems, and security concepts.
- **Professional communication:** Explaining technical information clearly and accurately.

## Skills To Highlight

- Security fundamentals
- Risk and control understanding
- Network basics
- Linux and SQL basics
- SIEM and log analysis concepts
- Incident response workflow
- Python automation basics
- Clear documentation
- Troubleshooting experience

## Study Documentation

Useful learning evidence can include:

- Study notes
- Short summaries
- Lab screenshots from safe environments
- Command examples
- Log review notes
- Incident report practice
- Risk register practice
- Reflection on lessons learned

## Interview Preparation Topics

- Explain the CIA triad.
- Describe risk, threat, and vulnerability.
- Explain phishing indicators.
- Describe how to triage a basic alert.
- Explain least privilege.
- Describe why logs are important.
- Explain what a SIEM does.

## Learning Plan

1. Finish the certificate modules.
2. Review terminology regularly.
3. Practice Linux, SQL, and Python basics.
4. Build comfort reading logs.
5. Practice explaining security concepts in simple language.
6. Continue with Security+ or SOC analyst study material.

## Quick Review

- Entry-level security learning should be organized and easy to explain.
- Clear notes help with revision and communication.
- Practical understanding matters more than memorizing definitions alone.
