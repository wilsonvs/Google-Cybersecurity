## Python Basics

- **Python:** Programming language commonly used for automation, scripting, and data handling.
- **Variable:** Stores a value.
- **Data type:** Kind of value, such as string, integer, float, Boolean, list, tuple, set, or dictionary.
- **Comment:** Note in code that explains what the code does.
- **Notebook:** Interactive coding environment often used for learning and testing.

## Control Flow

- **Conditional statement:** Runs code only when a condition is true.
- **Loop:** Repeats a block of code.
- **For loop:** Repeats over a sequence.
- **While loop:** Repeats while a condition remains true.

## Functions and Libraries

- **Function:** Reusable block of code.
- **Parameter:** Placeholder in a function definition.
- **Argument:** Actual value passed into a function.
- **Return statement:** Sends a value back from a function.
- **Library:** Collection of reusable code.
- **Module:** Python file or package with reusable functions.

Useful built-ins:

- `type()`
- `min()`
- `max()`
- `sorted()`
- `len()`

## Strings, Lists, and Regex

- **String:** Text data.
- **Index:** Position of an item in a sequence.
- **Substring:** Part of a string.
- **List:** Ordered collection of values.
- **Method:** Function attached to an object.
- **Regular expression:** Pattern used to search text.

Security examples:

- Find suspicious usernames.
- Search logs for IP addresses.
- Filter failed login attempts.
- Compare allowlists and blocklists.
- Extract indicators from text.

## Files and Automation

Python can automate tasks such as:

- Reading log files.
- Writing summaries.
- Updating allowlists.
- Searching for indicators.
- Filtering rows.
- Creating repeatable reports.

Basic file workflow:

1. Open file.
2. Read content.
3. Process or filter data.
4. Write updated output.
5. Close or safely handle file access.

## Errors and Debugging

- **Syntax error:** Code is written incorrectly.
- **Type error:** Operation uses the wrong data type.
- **Index error:** Code tries to access an item that does not exist.
- **Logic error:** Code runs but produces the wrong result.
- **Exception:** Error that occurs while code is running.
- **Debugger:** Tool used to inspect and fix code.

## Good Habits

- Use clear variable names.
- Keep code simple.
- Test with safe sample data.
- Avoid hardcoded secrets.
- Use comments where they explain logic.
- Follow consistent style.

## Quick Review

- Python helps automate repetitive security tasks.
- Lists, strings, regex, and files are especially useful for log work.
- Functions make code reusable.
- Debugging is part of normal scripting work.
