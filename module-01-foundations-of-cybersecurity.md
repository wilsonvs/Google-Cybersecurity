# Module 1: Foundations of Cybersecurity

## How to Use This Module

Use this module to build the base vocabulary and thinking style used throughout cybersecurity. The focus is not only memorizing terms, but understanding how security protects people, systems, data, and business operations.

Study each week in order. After each week, review the terms, answer the quick-check questions, and explain the ideas in your own words.

## Week 1: Introduction to Cybersecurity

### What You Should Understand

Cybersecurity is the practice of protecting systems, networks, devices, applications, and data from unauthorized access, misuse, disruption, or damage. A security analyst helps identify risks, monitor activity, investigate suspicious behavior, and support incident response.

The main goal is to reduce risk. Security does not mean making everything impossible to attack. It means using people, processes, and technology to make attacks harder, detect problems faster, and recover with less impact.

### Key Concepts

- Cybersecurity protects digital systems and information from threats.
- Information security focuses on protecting information in any form, including digital, paper, verbal, or physical.
- Security analysts support prevention, detection, investigation, response, and recovery.
- A strong security program uses policies, controls, monitoring, training, and incident response.
- Security work requires both technical skill and clear communication.

### Important Terms

- Asset - Anything valuable that needs protection, such as data, servers, laptops, accounts, or applications.
- Threat - Anything that can cause harm to an asset.
- Vulnerability - A weakness that can be exploited.
- Risk - The possibility that a threat will exploit a vulnerability and cause impact.
- Security posture - The overall strength and readiness of an organization's security.
- SOC - Security Operations Center; a team that monitors, detects, and responds to security events.
- SIEM - Security Information and Event Management; a tool used to collect and analyze logs.

### What To Practice

- Write examples of assets used by a small business.
- Match each asset with one possible threat and one possible vulnerability.
- Explain the difference between threat, vulnerability, and risk.
- Identify examples of cybersecurity in daily life, such as MFA, password managers, updates, and phishing filters.

### Quick Check

1. What is the main purpose of cybersecurity?
2. What is the difference between cybersecurity and information security?
3. Why is risk reduction more realistic than perfect security?
4. What does a SOC analyst usually monitor?
5. How are assets, threats, vulnerabilities, and risks connected?

## Week 2: Security History and Security Domains

### What You Should Understand

Cybersecurity has developed as technology has changed. Early computer security focused on protecting single systems. Modern cybersecurity must protect cloud platforms, mobile devices, remote users, applications, identity systems, and global networks.

Security domains help organize the field. They make it easier to understand the different areas that must work together to protect an organization.

### Key Concepts

- Security incidents have grown with the internet, cloud computing, mobile devices, and connected systems.
- Threat actors can include criminals, insiders, competitors, hacktivists, and nation-state groups.
- Security domains group related responsibilities such as access control, network security, asset protection, and incident response.
- A security analyst does not need to master every domain immediately, but should understand how the domains connect.

### Important Terms

- Threat actor - A person or group that can perform harmful activity against a system or organization.
- Insider threat - A risk caused by someone with authorized access, either intentionally or accidentally.
- Malware - Malicious software designed to damage, disrupt, spy, or gain unauthorized access.
- Social engineering - Manipulating people into sharing information or performing unsafe actions.
- Access control - Managing who can access systems, data, and resources.
- Security domain - A major area of cybersecurity knowledge and responsibility.

### Security Domains To Recognize

- Security and risk management
- Asset security
- Security architecture and engineering
- Communication and network security
- Identity and access management
- Security assessment and testing
- Security operations
- Software development security

### What To Practice

- Choose one real-world incident and identify which security domains were involved.
- List examples of insider threats, malware attacks, and social engineering attacks.
- Explain why access control is important in every organization.

### Quick Check

1. Why has cybersecurity become more complex over time?
2. What is a threat actor?
3. How can an insider threat happen by accident?
4. Why are security domains useful for learning cybersecurity?
5. Which security domain is most connected to SOC monitoring?

## Week 3: Frameworks, Controls, and Ethics

### What You Should Understand

Security frameworks provide organized guidance for managing cybersecurity risk. Controls are safeguards used to reduce risk. Ethics guide how security professionals handle access, information, privacy, and responsibility.

A security analyst may see sensitive logs, personal data, user activity, vulnerabilities, and internal incidents. That access must be handled responsibly.

### Key Concepts

- Frameworks give structure to security programs.
- Controls are specific safeguards that prevent, detect, or correct security problems.
- The CIA triad is a core model for thinking about information protection.
- Security professionals must follow laws, policies, and ethical standards.
- Privacy matters because cybersecurity often involves personal or sensitive information.

### Important Terms

- Confidentiality - Keeping information accessible only to authorized people or systems.
- Integrity - Keeping information accurate, complete, and trustworthy.
- Availability - Keeping systems and data accessible when needed.
- Control - A safeguard used to reduce risk.
- Preventive control - Stops or reduces the chance of an incident.
- Detective control - Identifies suspicious or harmful activity.
- Corrective control - Helps restore normal operations after a problem.
- PII - Personally identifiable information.
- PHI - Protected health information.
- Ethics - Principles that guide responsible behavior.

### Frameworks and References To Know

- NIST Cybersecurity Framework - Organizes security work around identify, protect, detect, respond, and recover.
- CIS Controls - Practical security controls for organizations.
- ISO 27001 - International standard for information security management.
- OWASP - Guidance for web application security risks.

### What To Practice

- Map examples of controls to preventive, detective, or corrective categories.
- Explain the CIA triad using a banking, healthcare, or school system example.
- Write a short explanation of why privacy is part of security work.

### Quick Check

1. What is the difference between a framework and a control?
2. What are the three parts of the CIA triad?
3. Why is MFA considered a security control?
4. What type of control is log monitoring?
5. Why do ethics matter for cybersecurity analysts?

## Week 4: Common Security Tools and Analyst Skills

### What You Should Understand

Security analysts use tools to collect evidence, monitor systems, analyze activity, and respond to incidents. Tools help, but the analyst still needs judgment, curiosity, documentation skill, and the ability to communicate clearly.

### Key Concepts

- SIEM tools centralize logs and alerts from many systems.
- IDS and IPS tools help detect or block suspicious network activity.
- Endpoint security tools monitor laptops, desktops, and servers.
- Vulnerability scanners help identify weaknesses that need fixing.
- Playbooks give repeatable steps for common security situations.
- Documentation is important because investigations must be clear and repeatable.

### Important Terms

- Log - A record of activity from a system, application, network device, or security tool.
- Alert - A notification that something may need investigation.
- IDS - Intrusion Detection System; detects suspicious activity.
- IPS - Intrusion Prevention System; can block suspicious activity.
- EDR - Endpoint Detection and Response; monitors endpoint behavior.
- Playbook - A step-by-step guide for responding to a specific scenario.
- False positive - An alert that appears suspicious but is not actually harmful.
- False negative - Harmful activity that is not detected.

### What To Practice

- Review example logs and identify timestamps, usernames, source IPs, destination IPs, and event types.
- Create a simple phishing investigation checklist.
- Compare SIEM, IDS, IPS, antivirus, and EDR at a basic level.
- Practice writing short incident notes using clear facts instead of assumptions.

### Quick Check

1. What does a SIEM help analysts do?
2. What is the difference between IDS and IPS?
3. Why are logs important in investigations?
4. What is a false positive?
5. Why should analysts document actions during an incident?
